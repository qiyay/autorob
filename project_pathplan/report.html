1. Depth-first search algorithm

The results for different scenes are shown below. It shows that depth-first search algorithm searched the entire column first, then took a step to the right column. The queue size was larger than other algorithms, so it's hard for dfs to find the to the goal. And the order for pushing unvisited neighbors to the queue matters, if push in a different direction, it may take longer time to find the path to same goal. 2. Breadth-first search algorithm

The results for different scenes are shown below. It shows see that in every iteration, breadth-first search algorithm followed the edge of a square. It visited more nodes than depth-first search algorithm, but it produced a shorter path than the dps. It will return the same length path if we exchange the start and the end.(Fig.6 & Fig.7)

3. Greedy-best-first search algorithm

The results for different scenes are shown below. It shows see that since the greedy best-first search algorithm only used the distance between the current node and goal node, the algorithm searched more rapidly if there was no obstacle between the start and goal node. But if there were obstacles, it spent more time for searching and did not guarantee the optimal path (Fig2).
